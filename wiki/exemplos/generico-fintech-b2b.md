# Exemplo de marca: Fintech B2B (genérico)

> ⚠️ **Não copie este arquivo como se fosse a sua marca.** É uma referência
> de como preencher [[../index|index.md]] e [[../conteudo/pov-da-marca]] para
> uma fintech B2B fictícia. Use como inspiração de formato, não de conteúdo.

## Sobre a empresa (exemplo)

- **Nome**: NomeFictícioPay
- **Setor**: fintech B2B — meios de pagamento para PMEs (10-200 funcionários)
- **ICP**: gestor financeiro ou COO de PME que opera com 50+ transações/dia e
  precisa conciliar múltiplos meios de pagamento.
- **Oferta principal**: plataforma única de conciliação financeira com API
  aberta para ERPs.
- **Diferencial competitivo**: conciliação em tempo real (não D+1) e SDK PT-BR
  com SLA de suporte para devs de cliente.
- **URL principal**: https://exemplo-fintech.com.br
- **Autor principal**: Engenheiro Sênior — 10 anos em meios de pagamento.
- **Tom de marca**: técnico, sóbrio, anti-buzzword.

## POVs proprietários (exemplo — 3 POVs)

### POV 1
**Posição**: Conciliação D+1 é dívida técnica disfarçada de feature, não
comodidade.

**Por quê**: PMEs operam com fluxo de caixa apertado; ver entrada de caixa
um dia depois compromete decisão de compra/pagamento. Players grandes
oferecem D+1 porque o produto deles foi construído antes da infraestrutura
permitir tempo real.

**Quem discordaria**: incumbentes e processadores tradicionais — vão dizer
que tempo real "não é necessário" e custa caro.

### POV 2
**Posição**: API e SDK são parte do produto, não complemento. Suporte para o
dev do cliente é obrigação, não cortesia.

**Por quê**: a integração é onde a maioria dos contratos morre. Sem
documentação PT-BR clara e SLA para o dev, mesmo o melhor produto perde.

**Quem discordaria**: fornecedores que tratam API como extensão e suporte de
integração como add-on premium.

### POV 3
**Posição**: PME não precisa de dashboard com 40 gráficos — precisa de 3
respostas claras.

**Por quê**: o gestor financeiro de PME não tem time de BI. Excesso de
visualização vira ansiedade, não informação.

**Quem discordaria**: produtos enterprise transplantados para PME sem
adaptação.

## Princípios derivados desses POVs

Quando esta marca escreve sobre "como escolher um meio de pagamento", ela
defende:
1. Prefira tempo real a D+1.
2. Avalie a documentação da API antes do preço.
3. Conte os campos de dashboard — menos é mais.

Esses são os 3 ângulos que entram no post toda vez. Sem POV, o post vira
listicle genérico que mil concorrentes já escreveram.

## Tom de voz aplicado

- ❌ "Nossa solução robusta vai empoderar a sua jornada financeira."
- ✅ "Conciliação em tempo real. API documentada. Três métricas no dashboard."

A diferença é o POV em ação.
